2단계: DOM interceptor 강화
==================================================

src/main.tsx의 기존 removeChild / insertBefore interceptor가
현재는 오류 위치만 main.tsx로 바꾸고 실제 노드 정보를 출력하지 못하고 있습니다.

기존 interceptor를 개선하여 호출 직전에 다음 정보를 반드시 출력하세요.

A. removeChild

고정 시작 문자열:
[DOM TRACE removeChild START]

각 항목을 개별 console.error로 출력:

- timestamp
- parent 노드 객체
- parent.nodeName
- parent.id
- parent.className
- parent.isConnected
- parent.parentNode
- child 노드 객체
- child.nodeName
- child.nodeType
- child.textContent 요약
- child.outerHTML 요약
- child.isConnected
- child.parentNode
- parent === child.parentNode
- parent.contains(child)
- Array.from(parent.childNodes)
- parent.outerHTML 요약
- document.activeElement
- location.pathname
- new Error("removeChild call stack").stack

오류가 실제로 발생할 조건인
child.parentNode !== parent
인 경우에는 별도 고정 문자열을 출력:

[DOM TRACE removeChild INVALID RELATION]

B. insertBefore

고정 시작 문자열:
[DOM TRACE insertBefore START]

각 항목을 개별 console.error로 출력:

- timestamp
- parent 노드 객체
- parent.nodeName
- parent.id
- parent.className
- parent.isConnected
- newNode 객체
- newNode.nodeName
- newNode.textContent 요약
- newNode.outerHTML 요약
- newNode.parentNode
- newNode.isConnected
- referenceNode 객체
- referenceNode?.nodeName
- referenceNode?.textContent 요약
- referenceNode?.outerHTML 요약
- referenceNode?.parentNode
- referenceNode?.isConnected
- referenceNode === null
- referenceNode?.parentNode === parent
- parent.contains(referenceNode)
- Array.from(parent.childNodes)
- parent.outerHTML 요약
- document.activeElement
- location.pathname
- new Error("insertBefore call stack").stack

referenceNode가 null이 아닌데 referenceNode.parentNode !== parent이면:

[DOM TRACE insertBefore INVALID REFERENCE]

주의사항:

- console.groupCollapsed을 사용하지 마세요.
- 로그 객체가 나중에 변경되어 보이지 않도록 문자열 스냅샷도 함께 남기세요.
- circular reference 때문에 JSON.stringify 오류가 나지 않도록 안전한 요약 함수를 사용하세요.
- 로그 출력 후 원래 DOM 메서드를 그대로 호출하세요.
- 오류를 무시하거나 return으로 우회하거나 try/catch로 삼키지 마세요.
- 인터셉터 자체가 앱 동작을 바꾸면 안 됩니다.
